# 噪声鲁棒的心音分割：生理合理性门控的混合解码

论文 *《Physiology-gated hybrid decoding for noise-robust heart-sound segmentation: a fair
benchmark of deep and classical models on CirCor》*（投稿目标：*Signal, Image and Video
Processing*, Springer）的**代码与结果**。论文手稿本身不托管在此——这个仓库是**可复现性包**：代码、复现论文每个数字的 `results.json`、以及配图。

---

## 一句话

把心音图（PCG）切成 4 个心动状态（S1 → 收缩期 → S2 → 舒张期）是下游杂音分析的前置。我们发现：

1. 在**公平、患者不重叠**的基准上，深度 1-D U-Net 在干净数据上明显胜过经典 Springer LR-HSMM（macro-F1 **0.877 vs 0.729**）。
2. 但在**带内噪声下深度模型崩溃**——而且崩得**自信**（校准误差 ECE 0.011 → 0.257）、崩得**生理上不可能**（心动周期违反率 0.06 → 0.22）。
3. 这个失败**无需标签即可测量**，于是我们把它变成解药：**免训练的混合解码器**把深度后验当作 HSMM duration-Viterbi 解码器的发射概率，再用 label-free 的违反率做门控。门控混合**对干净训练的深度模型 Pareto 占优**（干净不更差、有噪声大幅更好）。

> 诚实定调：这**不是**"经典打败深度"。干净数据上深度赢。价值在于**公平地刻画各自在哪失败**，以及一个在全信噪比范围内取两者之长的**免训练解码器**。

---

## 一、问题与缺口

深度 PCG 分割器在大规模真实数据集 **CirCor DigiScope 2022** 上报告了高精度，但有两个悬而未决的问题：

- **从未有同协议的公平对比**：深度 vs 经典 HSMM 在 CirCor 上没比过——此前数字来自不同数据集/切分/度量，根本不可比。
- **鲁棒性没人刻画**：深度分割器在听诊位置、杂音、尤其是采集噪声下的表现未知，而这些恰是临床每天都在变的条件。

我们填补这两点，并把发现变成方法。

## 二、方法

**(a) 公平基准**：1-D U-Net（1.86M 参数）vs Springer LR-HSMM，CirCor，患者不重叠切分，多种子置信区间，按 4 个听诊位置和有无杂音分层。

**(b) 失效机理**：带内噪声下深度模型**自信地错**（可靠性图 / ECE），且错误是**生理可见的**——心动周期**违反率**（一个 label-free 的量，衡量预测状态序列多频繁地打破 S1→收缩→S2→舒张的环序）随噪声单调上升。

**(c) 免训练混合 + 合理性门控**：把深度的每点后验 `q_t(s)` 复用为经典 HSMM duration-Viterbi 解码器的发射模型，从而**无需重训**就给深度输出施加生理时长与环序先验。门控只在深度输出不合理（违反率 > τ\*）时启用混合，τ\* 仅用**训练集干净**数据标定（τ\* = 0.20）。

## 三、结果

**干净基准（per-sample macro-F1，5 种子）**：深度全维度占优。

| | Deep U-Net | Springer HSMM |
|---|---|---|
| 总体 | **0.877 ± 0.007** | 0.729 ± 0.012 |

**带内白噪声鲁棒性（per-sample macro-F1）**：深度崩溃；混合在噪声下**同时超过两种单一方法**；门控保住干净精度。（深度/混合/门控 5 种子，HSMM 3 种子均值。）

| SNR | Deep | HSMM | Hybrid | Gated |
|---|---|---|---|---|
| 干净 | **0.867** | 0.729 | 0.849 | **0.867** |
| 10 dB | 0.827 | 0.697 | 0.831 | 0.830 |
| 5 dB | 0.747 | 0.657 | **0.790** | 0.758 |
| 0 dB | 0.474 | 0.580 | **0.686** | 0.587 |
| −5 dB | 0.048 | — | **0.472** | 0.276 |

- **失效特征**（0 dB）：ECE 0.011 → 0.257（置信 0.65、准确率仅 0.40）；周期违反率 0.06 → 0.22。
- **显著性**：配对 Wilcoxon（per-recording，跨种子 pooled，n = 1922）——门控 vs 深度 @0 dB，Δ = +0.098，p ≈ 6e-149；干净 Δ = +0.002，p = 0.33（统计无差异 → Pareto 占优）。
- **门控工作点**：在 5.0% 的干净录音上触发（95 分位设计），0 dB 升到 53.3%。

**泛化与诚实**：

| 检验 | 结果 |
|---|---|
| 真实 CirCor 采集噪声（0 dB） | 混合 0.663 > 深度 0.587、HSMM 0.556；门控保干净 0.868 |
| 更强 / 第二种架构（0 dB） | 宽 U-Net 0.44 → 混合 0.68；CNN-BiLSTM 0.42 → 混合 0.69 |
| 噪声增强训练（0 dB） | **0.801——强于免训练混合（0.686）**；我们坦白说明 |
| 跨语料 PhysioNet/CinC 2016（无标签） | 深度违反率 0.254；64% 超过 τ\* → 失效信号迁移成立 |

我们诚实定位混合方法：**当能用代表性噪声重训时，噪声增强更强**。混合的正交价值在于**免训练、模型无关**——可后挂到任何已部署的分割器上，无需重训。

## 仓库结构

| 路径 | 内容 |
|------|----------|
| `src/` | 训练、混合解码、评测、消融与配图脚本（PyTorch） |
| `figs/` | 配图（PDF/PNG） |
| `results.json` | 上文每个数字的单一可信源 |

**核心脚本**（`src/`）：`train_pcg.py` / `run_hybrid.py`（训 U-Net/CRNN；跨 SNR 评测深度/混合/门控/噪声增强）、`hybrid_decode.py`（深度后验发射 + duration-Viterbi；周期违反率）、`run_hsmm.py`（Springer LR-HSMM）、`eval_pcg.py`（macro-F1 + ±60ms 边界 F1）、`make_*.py`（配图与表）、`review_fixes.py`（门控触发率 + 训练集 τ\* 重标定）。

## 数据

**CirCor DigiScope 2022** 与 **PhysioNet/CinC 2016** 从 PhysioNet 获取，本仓库**不**二次分发。

## 复现

1-D U-Net：1.86M 参数（base 16、depth 4、5 类），AdamW（lr 1e-3、wd 1e-4），交叉熵，batch 32，30 epoch，4096 采样点（~4 s）窗口。切分均患者不重叠；深度/HSMM 为 5 种子、混合 3 种子均值（seed 0–4）。

> **注**：脚本使用开发服务器的绝对路径（如 `/data2/sjx/projects/pcg_seg`、conda 环境 `structecg`），运行前请改成你的环境。
